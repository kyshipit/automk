# AutoMLOS Tools Documentation

## Log Parser Tool

### Overview
The log_parser tool is a command-line utility for parsing binary log files generated by the autoMLOS system. It supports multiple output formats, intelligent message recognition, and flexible filtering options.

### Quick Start

1. **Build the tool** (if not already built):
```bash
./scripts/build_log_tools.sh
```

2. **Basic usage**:
```bash
./tools/log_parser/log_parser input.log output.txt
```

3. **View help**:
```bash
./tools/log_parser/log_parser --help
```

### Command Line Options

#### Output Format Control
```bash
-f, --format FORMAT    # Output format (text, json, csv, binary)
```
- `text`: Text format (default)
- `json`: JSON format for program processing
- `csv`: CSV format for Excel analysis  
- `binary`: Binary format preserving original structure

#### Database Configuration
```bash
-d, --db PATH          # Format database path (default: formats.db)
-t, --tags PATH        # Tag database path (default: tags.db)
```
- Format database contains message format definitions for intelligent parsing
- Tag database contains module tag definitions for log identification

#### Display Control
```bash
-c, --color MODE       # Color mode (never, minimal, always)
-s, --style STYLE      # Log style (smart, fixed, minimal, verbose)
```
- **Color modes**:
  - `never`: Disable all colors
  - `minimal`: Minimal colors (default)
  - `always`: Always enable colors

- **Log styles**:
  - `smart`: Intelligent formatting with auto-adjustment (default)
  - `fixed`: Fixed column width format
  - `minimal`: Minimal output
  - `verbose`: Detailed output with all information

#### Module Filtering
```bash
-m, --module TAG:LEVEL # Filter by module and level (e.g., CAN:INFO)
```
**Supported modules**:
- `SYSTEM`: System-level logs
- `KERNEL`: Kernel operations
- `DRIVER_CAN`: CAN driver logs
- `SERVICE_AI`: AI service logs  
- `SERVICE_DIAG`: Diagnostic service logs
- `APPLICATION`: Application logs

**Supported levels**:
- `DEBUG`: Debug information
- `INFO`: Informational messages
- `ERROR`: Error conditions
- `EMERGENCY`: Emergency situations

#### Verbose Mode
```bash
-v, --verbose          # Enable verbose output for debugging
```

### Usage Examples

#### Basic Parsing
```bash
# Text format with smart parsing
./tools/log_parser/log_parser system_log.bin output.txt

# JSON format for program processing
./tools/log_parser/log_parser system_log.bin output.json -f json

# CSV format for spreadsheet analysis
./tools/log_parser/log_parser system_log.bin output.csv -f csv
```

#### Module Filtering
```bash
# Filter CAN driver INFO level and above
./tools/log_parser/log_parser system_log.bin can_logs.txt -m DRIVER_CAN:INFO

# Filter multiple modules
./tools/log_parser/log_parser system_log.bin filtered.txt \
    -m DRIVER_CAN:INFO -m SERVICE_AI:ERROR

# System errors only
./tools/log_parser/log_parser system_log.bin system_errors.txt -m SYSTEM:ERROR
```

#### Display Customization
```bash
# No colors, minimal output
./tools/log_parser/log_parser system_log.bin output.txt -c never -s minimal

# Full colors, verbose output
./tools/log_parser/log_parser system_log.bin output.txt -c always -s verbose

# Fixed column format for alignment
./tools/log_parser/log_parser system_log.bin output.txt -s fixed
```

#### Advanced Usage
```bash
# Custom databases with verbose mode
./tools/log_parser/log_parser system_log.bin output.txt \
    -d custom_formats.db -t custom_tags.db -v

# JSON output with specific styling
./tools/log_parser/log_parser system_log.bin report.json \
    -f json -s verbose -m SYSTEM:INFO
```

### Build Scripts

#### build_log_tools.sh
One-click script for building and deploying the log parser tool.

```bash
# Full build and deployment
./scripts/build_log_tools.sh

# Clean build artifacts only
./scripts/build_log_tools.sh --clean

# Test deployed tool
./scripts/build_log_tools.sh --test

# View help
./scripts/build_log_tools.sh --help
```

### File Locations
- **Tool**: `./tools/log_parser/log_parser`
- **Format database**: `./tools/log_parser/formats.db`
- **Tag database**: `./tools/log_parser/tags.db`
- **Build script**: `./scripts/build_log_tools.sh`
- **Config script**: `./scripts/configure_log_parser.sh`

### Notes
- The tool automatically uses the databases in the same directory
- Multiple `-m` filters can be combined for complex filtering
- Color mode is automatically detected for terminal compatibility
- Binary format preserves the original log structure for advanced analysis
